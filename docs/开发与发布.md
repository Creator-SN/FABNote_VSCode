# FabNote 开发与发布说明

这份文档面向项目维护者，重点记录“怎么跑起来”和“怎么发版本”。

## 一句话说明

FabNote 本质上是一个基于 Power Editor 的 VS Code 插件：

- 扩展端：根目录，负责注册 `.fbn` 文件类型和 Custom Editor
- 前端端：`webview-ui/`，负责真正的可视化编辑器界面
- 构建产物：`webview-ui` 构建后输出到根目录 `media/`

## 启动流程

### 最常用方式：按 F5 调试

这个项目日常开发最常用的启动方式不是命令行，而是直接按 `F5`。

但有一个很重要的前提：如果你改的是 `webview-ui/` 里的前端代码，必须先 `yarn build`，再 `F5`。因为扩展实际加载的是根目录 `media/` 构建产物，不会直接读取 `src/`。

具体步骤：

1. 用 VS Code 打开项目根目录
2. 如果改过前端，先在 `webview-ui/` 执行 `yarn build`
3. 回到 VS Code 按 `F5`
4. 运行 `.vscode/launch.json` 里的 `Run Extension`
5. 新窗口 `Extension Development Host` 打开后，在里面测试 `.fbn` 文件

说明：

- `F5` 是“启动扩展调试”
- `yarn test` 是“跑自动化测试”
- 所以如果只是想看插件是否正常工作，优先用 `F5`
- 如果改过前端但没先 `yarn build`，`F5` 打开的通常还是旧界面

### 首次安装

根目录安装扩展依赖：

```bash
yarn
```

前端目录安装依赖：

```bash
cd webview-ui
yarn
```

### 日常开发

前端改完后先构建：

```bash
cd webview-ui
yarn build
```

然后回到项目根目录，在 VS Code 里按 `F5` 启动 `Run Extension`。

这个顺序不要反：

1. 先 `yarn build`
2. 再 `F5`

否则调试窗口里还是旧的 `media/` 内容。

### 调试时怎么验证

1. 在 Extension Development Host 中打开一个 `.fbn` 文件
2. 如果没有自动切到自定义编辑器，执行命令 `Preview FabNote`
3. 检查编辑、保存、另存是否正常

## 关键命令速查

### 扩展侧

```bash
yarn lint
yarn test
```

### 前端侧

```bash
cd webview-ui
yarn dev
yarn build
yarn preview
```

## 打包命令

推荐直接在项目根目录执行：

```bash
npx @vscode/vsce package
```

输出文件一般是：

```text
fabnote-<version>.vsix
```

例如仓库当前已有：

```text
fabnote-0.0.3.vsix
```

## 发布命令

```bash
npx @vscode/vsce publish
```

如果是首次在本机发布，先登录：

```bash
npx @vscode/vsce login CreatorSN
```

## 建议发布流程

1. 修改 `package.json` 版本号
2. 在 `webview-ui/` 执行 `yarn build`
3. 回根目录执行 `yarn lint`
4. 可选执行 `yarn test`
5. 执行 `npx @vscode/vsce package`
6. 确认 VSIX 可安装后再执行 `npx @vscode/vsce publish`

## 容易忘的点

- 真正被扩展加载的是根目录 `media/`，不是 `webview-ui/src/`
- 所以前端改动后如果没执行 `yarn build`，VS Code 里看到的通常还是旧页面
- `.vscode/launch.json` 已配置好调试入口，不需要额外写启动脚本
- 仓库当前没有内置 `package` / `publish` npm script，默认靠 `npx @vscode/vsce ...` 执行

## 关键文件

- [package.json](../package.json)：扩展元信息、命令、文件类型、发布者
- [extension.js](../extension.js)：扩展激活、自定义编辑器、保存逻辑
- [webview-ui/vite.config.js](../webview-ui/vite.config.js)：前端打包到 `media/` 的关键配置
- [.vscode/launch.json](../.vscode/launch.json)：F5 调试入口
